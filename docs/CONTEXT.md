# Project Vocabulary (CONTEXT.md)

Living domain glossary. Shared reference for all agents across all sessions.

**Purpose:** Canonical names and meanings for this game's concepts. Not a mechanics
spec — that lives in `design/gdd/`. This file answers "what do we call it?" so agents
stop inventing synonyms and design drift doesn't compound across sprints.

**Maintenance:** When a new term is established in a GDD or ADR, add it here.
When a term is renamed, update here first.
